# 1. NetworkManager

## Definition

NetworkManager is a Linux service that manages network interfaces and network connections. It simplifies configuring Ethernet, Wi-Fi, VPN, and other network connections through command-line and graphical tools.

In RHEL 8 and newer, **NetworkManager is the default network management service**. It provides management through tools such as `nmcli`, `nmtui`, GUI applications, and GNOME Settings. This matches the coverage in your uploaded module. :contentReference[oaicite:0]{index=0}

---

## Why do we use it?

Without NetworkManager, administrators would manually edit network configuration files and restart network services.

NetworkManager provides:

- Easy network configuration
- Static and DHCP support
- Multiple connection profiles
- Command-line automation
- GUI management
- Better scripting support

---

## Interview Answer

> NetworkManager is the default networking service in modern RHEL systems. It manages network interfaces and connection profiles. Instead of manually editing configuration files, I can use `nmcli` or `nmtui` to configure static IP addresses, DNS servers, gateways, and other network settings. It simplifies administration and is suitable for both interactive use and automation.

# 2. nmcli

## Definition

`nmcli` (NetworkManager Command Line Interface) is the command-line utility used to manage NetworkManager.

The training material highlights it as the preferred command-line tool when no graphical environment is available and for scripting. :contentReference[oaicite:1]{index=1}

---

## Why do we use it?

- Configure static IP addresses
- Configure DNS
- Bring interfaces up and down
- Create connection profiles
- Automate network configuration in scripts

---

## Common Commands

Display devices

```bash
nmcli device
```

Display active connections

```bash
nmcli connection show --active
```

Bring interface down

```bash
nmcli connection down enp0s3
```

Bring interface up

```bash
nmcli connection up enp0s3
```

---

## Configure Static IP

```bash
nmcli connection modify enp0s3 ipv4.addresses 192.168.1.100/24

nmcli connection modify enp0s3 ipv4.gateway 192.168.1.1

nmcli connection modify enp0s3 ipv4.dns 8.8.8.8

nmcli connection modify enp0s3 ipv4.method manual
```

Apply changes

```bash
nmcli connection down enp0s3

nmcli connection up enp0s3
```

These configuration steps are demonstrated in your uploaded networking module. :contentReference[oaicite:2]{index=2}

---

## Production Scenario

A new server is deployed with DHCP enabled, but the application requires a static IP.

Instead of editing configuration files manually, configure the interface using `nmcli`, then reactivate the connection.

---

## Interview Question

### Why use nmcli instead of editing configuration files?

Because `nmcli` communicates directly with NetworkManager, validates configuration, supports automation, and avoids manual editing mistakes.

`nmtui` (NetworkManager Text User Interface) provides a menu-driven interface for configuring networking from a terminal.

# 4. Interface Configuration

## Check Interface Status

```bash
ip address show
```

or

```bash
ip a
```

---

## Show Specific Interface

```bash
ip address show enp0s3
```

---

## Enable Interface

```bash
ip link set enp0s3 up
```

---

## Disable Interface

```bash
ip link set enp0s3 down
```

---

## Production Example

After replacing a network cable, verify that the interface is up before troubleshooting higher-layer issues.

# 5. Static vs DHCP

## Static IP

Advantages

- Fixed address
- Required for servers
- Predictable communication

Examples

- Database server
- Web server
- DNS server

---

## DHCP

Advantages

- Automatic configuration
- Easy administration
- Suitable for client machines

---

## Interview Question

Which should be used for production servers?

**Answer:** Static IP addresses, because services must always be reachable at a consistent address.

# 6. ifconfig vs ip

## Interview Question

Why is `ip` preferred over `ifconfig`?

### Answer

`ifconfig` is an older networking utility and is considered legacy on modern Linux distributions.

The `ip` command is part of the `iproute2` package and provides broader functionality, including interface management, routing, and address configuration.

---

## Common Commands

Display interfaces

```bash
ip a
```

Display routing table

```bash
ip route
```

Display links

```bash
ip link
```

# 7. ping

## Definition

`ping` checks whether a remote host is reachable by sending ICMP Echo Request packets.

---

## Syntax

```bash
ping google.com
```

Ping four packets

```bash
ping -c 4 google.com
```

---

## Production Use

- Verify connectivity
- Test gateway reachability
- Check Internet access
- Validate DNS resolution

---

## Troubleshooting

Cannot reach application?

Check in order:

```bash
ping gateway

ping 8.8.8.8

ping google.com
```

If 8.8.8.8 works but google.com fails, the issue is likely DNS.

---

# 8. ss

## Definition

`ss` displays socket and network connection information.

It is the recommended replacement for `netstat` on modern Linux systems.

---

## Common Commands

Listening ports

```bash
ss -tulpn
```

Established connections

```bash
ss -t
```

---

## Production Use

Verify whether a service is actually listening on the expected port.

Example:

```bash
ss -tulpn | grep 80
```

---

# 9. netstat

## Definition

`netstat` displays active network connections, routing tables, and interface statistics.

Although still seen on older systems, newer distributions recommend `ss`.

---

## Common Command

```bash
netstat -tulpn
```

---

# 10. tcpdump

## Definition

`tcpdump` captures network packets directly from an interface.

---

## Example

Capture packets on `enp0s3`

```bash
tcpdump -i enp0s3
```

Capture only port 80 traffic

```bash
tcpdump -i enp0s3 port 80
```

---

## Production Scenario

Application cannot communicate with another server.

Use `tcpdump` to confirm whether packets are leaving the source and arriving at the destination before assuming an application problem.

# Production Troubleshooting

## Server cannot access the network

### Step 1

Check interface

```bash
ip a
```

### Step 2

Check routes

```bash
ip route
```

### Step 3

Ping gateway

```bash
ping 192.168.1.1
```

### Step 4

Ping Internet

```bash
ping 8.8.8.8
```

### Step 5

Verify DNS

```bash
cat /etc/resolv.conf
```

### Step 6

Check listening ports

```bash
ss -tulpn
```

### Step 7

Capture packets

```bash
tcpdump -i enp0s3
```

---

# Best Practices

- Prefer `ip` over `ifconfig`.
- Prefer `ss` over `netstat`.
- Use `nmcli` for scripted network configuration.
- Verify connectivity from Layer 1 upward before troubleshooting applications.
- Use static IP addresses for production servers.
