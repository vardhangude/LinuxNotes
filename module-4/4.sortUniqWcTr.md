# 1. sort

## Interview Question

### What is the sort command?

### Answer

The `sort` command arranges the contents of a file or command output in ascending or descending order.

By default, it sorts alphabetically.

Syntax

```bash
sort filename
```

Example

```bash
sort names.txt
```

Before

```
John
Alex
David
Brian
```

After

```
Alex
Brian
David
John
```

---

## Common Options

### Reverse Sort

```bash
sort -r names.txt
```

---

### Numeric Sort

```bash
sort -n marks.txt
```

Example

Before

```
100
25
8
50
```

Without `-n`

```
100
25
50
8
```

With `-n`

```
8
25
50
100
```

---

### Sort by Specific Column

```bash
sort -k2 employees.txt
```

Sorts using the second column.

---

### Remove Duplicate Entries

```bash
sort -u file.txt
```

---

## Production Scenario

Find the top five memory-consuming processes.

```bash
ps aux | sort -nrk4 | head -5
```

Explanation

- `-n` → Numeric
- `-r` → Reverse
- `-k4` → Fourth column (Memory %)

---

## Interview Follow-up

Difference between

```bash
sort
```

and

```bash
sort -n
```

`sort` performs alphabetical sorting.

`sort -n` performs numeric sorting.

# 2. uniq

## Interview Question

### Explain uniq.

### Answer

The `uniq` command removes consecutive duplicate lines.

Syntax

```bash
uniq filename
```

Example

Before

```
apple
apple
banana
banana
orange
```

After

```
apple
banana
orange
```

---

## Important Note

`uniq` only removes **adjacent duplicates**.

If duplicate entries are scattered throughout the file, first sort the file.

```bash
sort file.txt | uniq
```

---

## Count Duplicate Lines

```bash
uniq -c file.txt
```

Output

```
4 ERROR
2 WARNING
1 INFO
```

---

## Display Only Duplicate Lines

```bash
uniq -d file.txt
```

---

## Display Only Unique Lines

```bash
uniq -u file.txt
```

---

## Production Scenario

Count repeated IP addresses.

```bash
cut -d' ' -f1 access.log | sort | uniq -c
```

Useful for identifying suspicious activity or DDoS attempts.

# 3. wc

## Interview Question

### Explain wc.

### Answer

`wc` stands for **Word Count**.

It displays:

- Number of lines
- Number of words
- Number of characters
- Number of bytes

---

## Count Lines

```bash
wc -l app.log
```

---

## Count Words

```bash
wc -w notes.txt
```

---

## Count Characters

```bash
wc -m notes.txt
```

---

## Count Bytes

```bash
wc -c notes.txt
```

---

## Production Scenario

Count the number of failed SSH login attempts.

```bash
grep Failed /var/log/secure | wc -l
```

Output

```
25
```

Meaning

Twenty-five failed login attempts were detected.

---

## Interview Follow-up

Difference between

```bash
wc -l
```

and

```bash
cat file | wc -l
```

Both return the same result.

However,

```bash
wc -l file
```

is more efficient because it avoids creating an unnecessary pipeline.

# 4. tr

## Interview Question

### What is tr?

### Answer

`tr` stands for **Translate**.

It replaces, deletes, or converts characters from standard input.

Unlike `sed`, `tr` works only with individual characters, not entire words.

---

## Convert Lowercase to Uppercase

```bash
echo "linux" | tr 'a-z' 'A-Z'
```

Output

```
LINUX
```

---

## Convert Uppercase to Lowercase

```bash
echo "DEVOPS" | tr 'A-Z' 'a-z'
```

Output

```
devops
```

---

## Delete Characters

```bash
echo "abc123" | tr -d '0-9'
```

Output

```
abc
```

---

## Replace Characters

```bash
echo "2026/07/22" | tr '/' '-'
```

Output

```
2026-07-22
```

---

## Compress Repeated Characters

```bash
echo "aaabbbcccc" | tr -s 'a-z'
```

Output

```
abc
```

---

## Production Scenario

Convert application logs to uppercase before comparison.

```bash
cat app.log | tr 'a-z' 'A-Z'
```

# Real Production Example

## Interview Question

Application is slow.

Logs contain thousands of entries.

How will you identify the most common error?

### Answer

Step 1

Extract error messages.

```bash
grep ERROR app.log
```

Step 2

Sort the entries.

```bash
grep ERROR app.log | sort
```

Step 3

Count duplicate errors.

```bash
grep ERROR app.log | sort | uniq -c
```

Step 4

Display the most frequent errors first.

```bash
grep ERROR app.log | sort | uniq -c | sort -nr
```

This quickly identifies recurring issues without manually reading the entire log.

# Best Practices

✅ Use `sort` before `uniq` when duplicates are not adjacent.

✅ Use `wc -l` directly on files when possible instead of piping through `cat`.

✅ Use `tr` only for character-level transformations.

✅ Use pipelines to reduce manual effort and improve efficiency.

---

# Common Mistakes

❌ Assuming `uniq` removes all duplicates without sorting.

❌ Using `tr` to replace complete words.

❌ Forgetting `-n` while sorting numbers.

❌ Using `cat file | wc -l` unnecessarily.
