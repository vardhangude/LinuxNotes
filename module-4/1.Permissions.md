# 1. Linux File Permissions

## Interview Question

### Explain Linux File Permissions.

### Answer

Linux follows a permission-based security model where every file and directory has three permission categories.

- Owner (User)
- Group
- Others

Each category can have

- Read (r)
- Write (w)
- Execute (x)

Example

```bash
-rwxr-xr--
```

Meaning

| Section | Permission |
|----------|------------|
| Owner | Read, Write, Execute |
| Group | Read, Execute |
| Others | Read |

Numeric representation

| Permission | Value |
|------------|------|
| Read | 4 |
| Write | 2 |
| Execute | 1 |

Examples

| Octal | Meaning |
|------|---------|
|755|rwx r-x r-x|
|750|rwx r-x ---|
|644|rw- r-- r--|
|600|rw-------|

### Interview Follow-up

Why do directories require Execute permission?

Execute permission on a directory allows users to enter (traverse) the directory. Without execute permission, even if read permission exists, users cannot access files inside it.

# 2. chmod

## Interview Question

### Explain chmod.

### Answer

The chmod command changes file or directory permissions.

Numeric Mode

```bash
chmod 755 script.sh
chmod 644 config.conf
chmod -R 755 /opt/app
```

Symbolic Mode

```bash
chmod u+x script.sh

chmod g-w file

chmod o-r file

chmod a+x script.sh
```

### Production Scenario

Jenkins deployment failed.

Error

```
Permission denied
```

Investigation

```bash
ls -l deploy.sh
```

Output

```
-rw-r--r--
```

Fix

```bash
chmod +x deploy.sh
```

Now Jenkins can execute the deployment script.

---

### Interview Follow-up

What does chmod 755 mean?

Owner

```
rwx
```

Group

```
r-x
```

Others

```
r-x
```

---

### Why is chmod 777 discouraged?

Giving everyone write permission creates serious security risks.

Anyone can

- Modify files
- Delete files
- Replace scripts
- Execute malicious code

In production, always follow the **Principle of Least Privilege**.

# 3. chown

## Interview Question

Explain chown.

### Answer

The chown command changes the ownership of a file or directory.

Examples

```bash
chown nginx index.html

chown nginx:nginx index.html

chown -R appuser:developers /opt/application
```

---

### Production Example

Nginx cannot read website files.

Check

```bash
ls -l /var/www/html
```

Incorrect owner

```
root root
```

Fix

```bash
chown -R nginx:nginx /var/www/html
```

Restart service.

Problem solved.

---

### Interview Follow-up

Difference between chmod and chown?

| chmod | chown |
|---------|--------|
| Changes permissions | Changes owner |

# 4. chgrp

## Interview Question

Explain chgrp.

### Answer

Changes only the group ownership.

Example

```bash
chgrp developers app.log
```

Verify

```bash
ls -l app.log
```

---

### Interview Follow-up

Difference between chgrp and chown?

```
chown

Changes owner
Can also change group

chgrp

Changes only group
```

# 5. SUID

## Interview Question

Explain SUID.

### Answer

SUID (Set User ID) allows an executable to run using the permissions of its owner instead of the user executing it.

Permission value

```
4000
```

Example

```bash
chmod 4755 program
```

Verify

```bash
ls -l program
```

Output

```
-rwsr-xr-x
```

Notice

```
s
```

instead of

```
x
```

---

### Real Example

```bash
ls -l /usr/bin/passwd
```

passwd runs as root.

Otherwise users could never update their own passwords.

---

### Production Usage

Used only when applications require temporary elevated privileges.

Improper SUID permissions can become security vulnerabilities.

# 6. SGID

## Interview Question

Explain SGID.

### Answer

SGID (Set Group ID)

Permission value

```
2000
```

Directories

Every new file inherits the parent directory's group.

Example

```bash
chmod 2775 /shared
```

Production Use

Developer teams sharing source code.

All newly created files automatically belong to the developers group.

# 7. Sticky Bit

## Interview Question

Explain Sticky Bit.

### Answer

Sticky Bit prevents users from deleting files owned by other users.

Permission value

```
1000
```

Example

```bash
chmod 1777 /shared
```

Common example

```bash
ls -ld /tmp
```

Output

```
drwxrwxrwt
```

Notice

```
t
```

---

### Production Usage

Shared directories

```
/tmp

/upload

/shared
```

# 8. umask

## Interview Question

Explain umask.

### Answer

umask defines the default permissions for newly created files and directories.

Check

```bash
umask
```

Example

```
022
```

Default Result

Files

```
644
```

Directories

```
755
```

Set

```bash
umask 027
```

Now

Files

```
640
```

Directories

```
750
```

---

### Interview Follow-up

Difference between chmod and umask?

```
chmod

Changes existing permissions

umask

Defines permissions for newly created files only
```

# 9. Access Control Lists (ACL)

## Interview Question

Why do we use ACL?

### Answer

Traditional Linux permissions support

- Owner
- Group
- Others

Suppose one QA engineer needs write access to a file owned by the Developers group.

Changing ownership isn't appropriate, and changing the group could affect many users.

ACL allows granting permissions to specific users or groups without affecting the existing permission model.

---

### View ACL

```bash
getfacl report.txt
```

---

### Add ACL

```bash
setfacl -m u:john:rwx report.txt
```

---

### Group ACL

```bash
setfacl -m g:developers:r-x report.txt
```

---

### Default ACL

```bash
setfacl -d -m g:developers:rwx /project
```

Every new file inside `/project` inherits these permissions.

---

### Remove ACL

```bash
setfacl -b report.txt
```

---

### Verify ACL

```bash
ls -l
```

Example

```
-rw-rw-r--+
```

The `+` indicates additional ACL entries.

# Production Troubleshooting

## Scenario

Application displays

```
Permission denied
```

### Troubleshooting Steps

Check permissions

```bash
ls -l file
```

Check ownership

```bash
id appuser
```

Check ACL

```bash
getfacl file
```

Check parent directory permissions

```bash
ls -ld /parent
```

If SELinux is enabled

```bash
getenforce
```

Review SELinux contexts before changing permissions.

# Best Practices

✅ Never use `chmod 777` as the first solution.

✅ Follow the Principle of Least Privilege.

✅ Use groups for team access.

✅ Use ACL only when finer-grained permissions are required.

✅ Verify changes with `ls -l` and `getfacl`.
